# EAT.Z: The Future of Campus Dining
## Project Presentation

---

## 1. Introduction
**EAT.Z** is a next-generation food delivery platform designed specifically for closed-loop environments like campuses, tech parks, and large office complexes. 

**The Problem:**
*   Confusing delivery addresses in large complexes.
*   Inefficient meetup points.
*   Outdated, clunky user interfaces.

**The Solution:**
*   **Pinpoint Delivery**: Deliver to specific, pre-defined shelters/landmarks.
*   **Gen Z Aesthetic**: A modern, engaging, and fluid user experience.
*   **Seamless Ecosystem**: Integrated flows for Customers, Merchants, and Admins.

## 3. The "Pinpoint" Advantage
Unlike traditional apps that rely on vague GPS pins or street addresses, **EAT.Z** uses a **Master Location System**.

*   **Admins** define valid pickup points (Shelters).
*   **Customers** select from this curated list.
*   **Merchants** know exactly where to deliver.
*   **Result**: Zero confusion, faster deliveries, happier users.

## 4. Technology Stack
*   **Frontend**: React.js (Vite) for a blazing fast SPA experience.
*   **Styling**: Vanilla CSS with CSS Variables for a custom, high-performance design system (Glassmorphism, Neon accents).
*   **State Management**: React Context API for global state (User, Cart, Data).
*   **Icons**: Lucide-React for consistent, modern iconography.
*   **Charts**: Recharts for data visualization.
